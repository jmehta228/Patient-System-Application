# Database Population Tool Guide

## Overview
The auto-populate feature has been implemented as a **separate developer/admin utility** that is **NOT exposed in the user interface**. This keeps the production application clean while providing a powerful tool for testing and development.

---

## Files Created

### 1. PatientDataGenerator.java
**Location**: `src/ButtonActionFiles/PatientDataGenerator.java`
- Core utility class for generating random patient data
- 150+ realistic first names, 140+ last names
- Configurable age ranges
- Can be used programmatically in other tools

### 2. AutoGeneratePatients.java  
**Location**: `src/ButtonActionFiles/AutoGeneratePatients.java`
- UI dialog (kept for potential future admin panel use)
- **NOT integrated into main application UI**
- Can be used standalone if needed

### 3. TestAutoGenerate.java
**Location**: `src/TestAutoGenerate.java`
- **Primary tool for database population**
- Standalone command-line utility
- Interactive menu system
- Safe and easy to use

---

## How to Use the Population Tool

### Method 1: Interactive Command-Line Tool (Recommended)

Run the standalone tool:

```bash
java -cp "out/production/Patient-System-Application:lib/*" TestAutoGenerate
```

**Features:**
- Shows current database status
- Interactive menu with 5 options:
  1. Generate patients with default age range (0-100)
  2. Generate patients with custom age range
  3. Quick test - Generate 10 sample patients
  4. Show sample data (without inserting)
  5. Exit

**Example Session:**

### Method 2: Programmatic Usage

You can also use the `PatientDataGenerator` class directly in your own code:

```java
import ButtonActionFiles.PatientDataGenerator;

// Generate 50 random patients (ages 0-100)
int count = PatientDataGenerator.generateAndInsertPatients(50);
System.out.println("Inserted: " + count + " patients");

// Generate 25 adult patients (ages 18-65)
count = PatientDataGenerator.generateAndInsertPatients(25, 18, 65);

// Generate 15 elderly patients (ages 65-90)
count = PatientDataGenerator.generateAndInsertPatients(15, 65, 90);

// Just generate random data without inserting
String firstName = PatientDataGenerator.generateFirstName();
String lastName = PatientDataGenerator.generateLastName();
String birthdate = PatientDataGenerator.generateBirthdate();
```

## Data Format

All generated patients have:
- **PatientFirstName**: Random realistic first name
- **PatientLastName**: Random realistic last name  
- **PatientBirthdate**: Random date in MM/dd/yyyy format
- **PatientStatus**: Always "Sick" (can be changed via Transfer Patient feature)
- **PatientID**: Auto-generated by database

---

## Important Notes

✅ **Separated from User Interface**
   - Users cannot accidentally populate the database
   - Keeps production app clean and professional
   - Only accessible to developers/admins via command line

✅ **Safe to Use**
   - Maximum 1000 patients per operation
   - Shows confirmation before inserting
   - Reports success/failure counts
   - Non-destructive (only adds, never deletes)

✅ **Flexible**
   - Can specify exact age ranges
   - Can generate small or large datasets
   - Can preview data without inserting
   - Can be integrated into test scripts

## Technical Details

- **Language**: Java
- **Database**: MySQL via JDBC
- **Thread Safety**: Random generation is thread-safe
- **Performance**: Batch inserts for efficiency
- **Error Handling**: Catches and reports all errors
- **Validation**: Built-in age and count validation
