# Auto-Generate Patients Feature

## Overview
Added functionality to automatically generate random patient data for testing and demonstration purposes.

## New Files Created

### 1. PatientDataGenerator.java
**Location**: `src/ButtonActionFiles/PatientDataGenerator.java`

**Purpose**: Utility class for generating random patient information

**Key Features**:
- Generates realistic first names (150+ options)
- Generates realistic last names (140+ options)
- Creates random birthdates within specified age ranges
- Configurable age range (default: 0-100 years)
- Batch generation support

**Main Methods**:
```java
// Generate individual components
String generateFirstName()
String generateLastName()
String generateBirthdate()
String generateBirthdate(int minAge, int maxAge)

// Generate and insert into database
int generateAndInsertPatient()
int generateAndInsertPatients(int count)
int generateAndInsertPatients(int count, int minAge, int maxAge)
```

### 2. AutoGeneratePatients.java
**Location**: `src/ButtonActionFiles/AutoGeneratePatients.java`

**Purpose**: Modern UI dialog for auto-generating patients

**Features**:
- Clean, modern interface matching the app theme
- Input fields for:
  - Number of patients (1-1000)
  - Minimum age (optional, default: 0)
  - Maximum age (optional, default: 100)
- Real-time validation
- Progress feedback during generation
- Toast notifications for success/error
- Background processing using SwingWorker
- Auto-closes after successful generation

## Usage

### From Sidebar Navigation
1. Click "Auto-Generate" in the sidebar menu
2. Dialog opens with form fields
3. Enter number of patients (required)
4. Optionally specify age range
5. Click "Generate" button
6. Patients are created in background
7. Success notification appears
8. Dialog closes automatically

### From Dashboard Quick Actions
1. On dashboard, find "Quick Actions" section
2. Click "Auto-Generate" card
3. Same dialog flow as above

### Programmatic Usage
```java
// Generate single patient
PatientDataGenerator.generateAndInsertPatient();

// Generate 50 patients with default age range (0-100)
int count = PatientDataGenerator.generateAndInsertPatients(50);

// Generate 25 patients aged 18-65
int count = PatientDataGenerator.generateAndInsertPatients(25, 18, 65);
```

## Data Format

All generated patients:
- **PatientFirstName**: Random from 150+ common first names
- **PatientLastName**: Random from 140+ common last names
- **PatientBirthdate**: Random date in MM/dd/yyyy format
- **PatientStatus**: Always "Sick" (can be transferred to "Recover" later)
- **PatientID**: Auto-generated by database

## Benefits

1. **Testing**: Quickly populate database for testing features
2. **Demonstrations**: Show full UI with realistic data
3. **Performance Testing**: Test with various data volumes
4. **UI Development**: View how interface handles different data sizes
5. **Time Saving**: No need to manually add test patients

## Validation

The UI validates:
- Number of patients is required and between 1-1000
- Ages must be non-negative integers
- Minimum age cannot exceed maximum age
- All inputs must be valid numbers

## Error Handling

- Invalid inputs show error toast notifications
- Failed insertions are logged to console
- Partial success shows warning with count
- Button disabled during generation to prevent duplicates
- All database errors caught and reported

## Technical Details

- Uses `SwingWorker` for async database operations
- Thread-safe random generation
- Efficient batch processing
- Modern Material Design-inspired UI
- Fully integrated with existing theme system
- Toast notifications for user feedback
